# arXiv 논문 게시 날짜 메커니즘 분석

## 📅 arXiv 날짜 시스템의 핵심 특성

### 1. Submitted vs 게시일의 차이
- **Submitted**: 연구자가 논문을 arXiv에 제출한 실제 날짜
- **게시일**: arXiv가 실제로 논문을 공개하는 날짜
- **중요**: 제출일과 게시일은 일치하지 않음

### 2. arXiv 게시 스케줄
- **게시 요일**: 미국 동부시간 기준 일요일~목요일만
- **휴무일**: 금요일, 토요일, 미국 연방공휴일에는 게시 안 함

### 3. 게시 지연 현상
연구자가 금요일, 토요일, 공휴일에 논문을 제출해도, arXiv는 다음 유효 게시일(일요일~목요일)까지 기다린 후 공개

- **검색일**: 2025-09-02 (월요일) 오전 11:16 AM

#### 2025-09-03 검색 결과  
- **검색일**: 2025-09-03 (화요일)
- **발견된 논문**:
  - 2025-09-02 Submitted 논문 ✅
  - 2025-09-01 Submitted 논문 ✅ (노동절 제출)
  - 2025-08-31 Submitted 논문 (토요일 제출?)
  - 2025-08-30 Submitted 논문 (금요일 제출?)

### 🧩 메커니즘 추론

#### 게시 지연 패턴
```
제출일 → 게시일 매핑 (2025년 9월 사례)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
8월 30일 (금) 제출 → 9월 3일 (화) 게시
8월 31일 (토) 제출 → 9월 3일 (화) 게시  
9월 1일 (일/공휴일) 제출 → 9월 3일 (화) 게시
9월 2일 (월) 제출 → 9월 3일 (화) 게시
```

#### 게시 일정 규칙
1. **당일 게시 불가**: 오늘 제출된 논문은 당일에 절대 검색되지 않음 (최소 1일 후 게시)
2. **금요일 제출** → 다음 주 월요일 또는 화요일 게시
3. **토요일 제출** → 다음 주 월요일 또는 화요일 게시
4. **공휴일 제출** → 다음 유효 게시일에 게시
5. **평일 제출** → 다음날 또는 다음 유효 게시일에 게시

## 💡 수집 전략에 미치는 영향

### 현재 수집 로직의 문제점
1. **누락 가능성**: 지연 게시된 논문 놓칠 수 있음
2. **중복 수집**: 같은 논문을 여러 번 수집할 가능성
3. **시간대 혼란**: 한국 시간 vs 미국 시간 기준 선택

### 권장 수집 전략
```
최적 전략: "가장 최근 수집 논문 날짜부터 오늘까지"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 마지막 수집된 논문의 Submitted 날짜 확인
2. 해당 날짜부터 현재까지 전체 범위 검색  
3. 중복 제거는 URL 기반으로 자동 처리
4. 지연 게시된 논문도 자동으로 포함
```

### 🎯 핵심 인사이트
- **Submitted 날짜**: 실제 연구자 제출일 (정확)
- **게시 타이밍**: arXiv 내부 스케줄에 따라 지연
- **당일 게시 불가**: **arXiv는 오늘 제출된 논문을 당일에 게시하지 않음** (최소 1일 지연)
- **수집 전략**: 넓은 날짜 범위로 안전하게 수집 후 중복 제거가 최적
