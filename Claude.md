# arXiv 논문 메타데이터 수집기 - 개발 지침서

## 프로젝트 개요
**프로젝트명**: arXiv 논문 메타데이터 수집기  
**목적**: AI/ML 분야 최신 논문들의 메타데이터를 자동으로 수집하고 저자 이력을 추적하는 시스템 개발

---

## 🎯 수집 목표 데이터

### 기본 메타데이터 - 각 column의 데이터 예시 format을 기술해야 됨
- **논문 제목** (Title)
- **저자 정보** (Authors)
- **소속 기관** (Organization)
- **논문 초록** (Abstract)
- **주제 분야** (Subjects)
- **제출일/발표일** (Submitted/Announced dates)
- **arXiv URL** 및 원본 HTML 데이터
- **License**
- **각 저자의 소속 기관** Optional


### 고급 추적 데이터
- **저자 이력 추적**: 과거 소속 기관 변화 이력
- **빅테크 경력**: 현재 다른 기관에 있더라도 과거 대기업 소속 이력이 있는 연구자 식별
  - 목표: 연구 성과와 기업 경험의 상관관계 분석

## 📊 최종 출력 형식

### CSV 파일 구조
**파일명**: `results/1_URL_of_paper_abstractions.csv`

**컬럼 구성**:
- `Collected_date` [YYYY-MM-DD HH:MM:SS]: 데이터 수집 시간
- `Arxiv_abs_url`: 논문의 arXiv URL
- `Title`: 논문 제목
- `Authors`: 저자 목록 (세미콜론으로 구분)
- `Submitted`: 논문 제출일 [YYYY-MM-DD]
- `Originally_announced`: 논문 발표일
- `Subjects`: 주제 분야 (세미콜론으로 구분)
- `Organization`: 소속 기관 정보 (NULL 허용) 🚧 **개발 예정**
- `Abstract`: 논문 초록 🚧 **개발 예정**
- `Raw_HTML`: 파싱되지 않은 원본 HTML 데이터 🚧 **개발 예정**

## 🎯 특별 요구사항

### 저자 이력 추적 시스템
**핵심 아이디어**: 현재 학계나 스타트업에 있는 연구자라도 과거에 빅테크 기업 (Google, Microsoft, Meta, Amazon, Apple 등) 경험이 있다면 그 성과를 특별히 추적

**구현 방향**:
1. 저자별 논문 이력 데이터베이스 구축
2. 각 논문별 저자 소속 기관 수집
3. 시간순 소속 변화 추적
4. 빅테크 경력 플래그 및 기간 기록

**기대 효과**:
- 기업 연구 경험이 학계 연구에 미치는 영향 분석
- 연구자 이동 패턴 파악
- 산학 연계 효과 정량적 측정
